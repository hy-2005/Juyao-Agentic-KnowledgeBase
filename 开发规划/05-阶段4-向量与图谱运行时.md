## 本阶段技术栈

| 类别 | 选型 |
|------|------|
| 语义召回 | **Qdrant** + 阶段 2 检索管线（含混合时带 **ES**） |
| 图查询 | **Neo4j**；**Cypher**（**Text-to-Cypher**，模板或 schema 约束防注入） |
| 编排 | **LangChain**；路由 / 合并生成 LLM **`glm-5`** |
| 策略 | **方案 1**：向量先行，复杂关系再走图；失败降级纯向量 |

## 开发目标

1. 落地 **「向量先行，图补深度」**：默认向量召回打底；对需要多跳推理的问题再走 Neo4j。
2. 运行时路由：由 LLM 判断「科普/语义即可」vs「需要关系链/禁忌/并发症等多跳」。
3. 图谱查询：**Text-to-Cypher**（带 schema 约束与白名单，防注入）；失败则**降级为纯向量**（见主文档降级策略）。
4. 生成阶段：将「向量检索上下文 + 图查询结果」合并，输出带溯源的最终回答。

## 交付物

- 端到端 Demo：至少覆盖三类问题——纯语义、两跳关系、更长路径（如并发症/关联用药脉络）。
- Cypher 生成模板或受限生成规则说明。
- 图谱异常或无结果时的降级路径日志可追溯。

## 验收要点

- 复杂关系类问题答案中能体现结构化关系，且仍引用原文 chunk。
- 图查询失败不影响整体可用性（退回向量）。

## 依赖与风险

- **依赖**：阶段 2、阶段 3。
- **风险**：Cypher 幻觉；必须用 schema、示例 few-shot、执行前校验或仅允许参数化查询模板。
