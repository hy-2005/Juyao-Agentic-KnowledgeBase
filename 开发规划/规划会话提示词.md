# 规划会话提示词（新开窗口时使用）

把下面 **「可复制提示词」** 整段复制到新会话即可恢复上下文；详细约定见本文后半部分与本目录其余 md。

---

## 可复制提示词（粘贴到新会话首条消息）

```
你在本仓库协作「通用超级知识库 / 医疗科普向 Agentic RAG」的开发规划。

约定：
1. 正式规划在文件夹 `开发规划/`：`README.md` 为总览；`01`～`07` 为按顺序的阶段文档（阶段 0→6 对应文件 01→07）。
2. 阶段文档是唯一真相来源：技术约定、目标、交付物直接写在对应 `0x-阶段x-*.md` 里；**不要**让阶段 md 依赖或引用 `人工规划.md`。
3. `人工规划.md` 仅是个人备忘（记录事后觉得不合理的需求等），**不参与**阶段文档的依赖链。
4. 每个阶段文档标题下有「执行状态」：`当前` 取值限定为 未开始 / 进行中 / 已完成 / 阻塞 / 跳过；`更新记录` 写日期与简述。推进时请同步更新 `README.md` 里的「一览」表（若有）。
5. 当前 chunk 切分等业务默认值写在 `01-阶段0-工程骨架与数据公约.md`，勿凭空假设未写入文档的模型或栈。
6. 技术栈总表见 `开发规划/技术栈约定.md`；各阶段还有「本阶段技术栈」表。当前已定：`glm-5`（主 LLM，OpenAI 兼容 API）、`mxbai-embed-large:latest`（Embedding）、`jina-reranker-v2`（重排）；其余待定项见该文件第七节。
7. 业务背景与方案对比见仓库根目录 `通用超级知识库项目.md`。

若用户要求改规划：直接编辑对应阶段 md（及必要时 README 一览）；若用户问进度：先读各阶段「执行状态」与 README 一览。
```

## 目录与文件分工（给助手看的说明）

| 路径 | 用途 |
|------|------|
| `通用超级知识库项目.md` | 背景、技术栈、RAG/图/Agent 方案对比、降级思路 |
| `开发规划/README.md` | 总览、执行状态取值说明、各阶段一览表、文档索引 |
| `开发规划/技术栈约定.md` | 全栈选型、待定项清单；与各阶段「本阶段技术栈」配套 |
| `开发规划/01-阶段0-工程骨架与数据公约.md` | 流水线、chunk_id 公约、**当前 chunk 切分约定**（如 Ollama + qwen2:1.5b、重叠兜底） |
| `开发规划/02`～`07` | 阶段 1～6 的目标、交付、验收、执行状态 |
| `开发规划/人工规划.md` | 个人备忘，**不被阶段文档引用** |

## 阶段编号与文件名对照

| 执行顺序 | 文件名 | 内容概要 |
|----------|--------|----------|
| 1 | `01-阶段0-…` | 工程骨架、数据公约、chunk 管线 |
| 2 | `02-阶段1-…` | 基础 RAG MVP |
| 3 | `03-阶段2-…` | 检索增强 |
| 4 | `04-阶段3-…` | 知识图谱构建（离线） |
| 5 | `05-阶段4-…` | 向量 + 图谱运行时（方案 1） |
| 6 | `06-阶段5-…` | Agentic 与降级 |
| 7 | `07-阶段6-…` | 评测、运维、方案 2 预留 |

## 修改规划时的注意点

- 变更技术约定：**改对应阶段 md**，不要新建「第二套」说法分散在别处。
- 更新进度：**改该阶段「执行状态」**，并尽量同步 `README.md` 一览表。
- 不要把 `人工规划.md` 写进阶段文档的正文依赖（用户明确要求解耦）。
