## 本阶段技术栈

| 类别 | 选型 |
|------|------|
| RAG 编排 | **LangChain** |
| 向量库 | **Qdrant（本地）**（写入、检索、元数据带 `chunk_id`） |
| Embedding | **Ollama**：`mxbai-embed-large:latest` |
| 生成 / 聊天 LLM | **`glm-5`**（OpenAI 兼容 API，拒答与免责放在系统提示词） |
| 终端形态 | CLI / 简易 Web / HTTP API **三选一或组合**（Demo） |

## 开发目标

1. 实现「用户提问 → 检索相关 chunk → LLM 生成回答」，并附带**引用片段**。
2. **Chunk 来源**：与阶段 0 一致——**Ollama `qwen2:1.5b` 语义切分** + **前后段重叠兜底**；本阶段不擅自改为固定字数切分（若调整切分策略，请直接改 **01-阶段0** 并同步重灌数据）。可选在之上叠加 Parent-Child（父摘要 + 子检索）作为后续优化项，不以替换上述策略为前提。
3. **防幻觉**：系统提示词约束 + **检索质量不足时拒答或弱答**（先不做独立校验 LLM）。
4. 产品红线：回答模板中包含**免责声明**（科普参考、不替代诊疗），与主文档一致。

## 交付物

- 问答入口一种即可：CLI / 简易 Web / API。
- Qdrant（或既定向量库）完成端到端写入与检索。
- 固定 **小测试集**（如 20～50 条）用于人工过目。
- 基础系统提示词与拒答策略文档化。

## 验收要点

- 能覆盖「常见科普类」问题的语义检索与流畅回答。
- 每条回答可指向引用 chunk；低相似度场景行为符合预设（拒答/提示无法确认）。
- 展示免责声明与「非诊疗」边界。

## 依赖与风险

- **依赖**：阶段 0（含其中约定的切分策略）。
- **风险**：`qwen2:1.5b` 语义边界判断误差会导致 chunk 粒度波动；依靠重叠与阶段 2 检索增强缓解。若效果差，优先在阶段 2 补救，或**修订 01-阶段0** 中的切分模型/参数并重灌，而非过早上图谱。
