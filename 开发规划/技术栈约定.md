# 技术栈约定（全项目）

本文与根目录 [通用超级知识库项目.md](../通用超级知识库项目.md) 一致；**已定项**直接按此实现。**模型名变更**请同步更新本文件与受影响阶段 md。

---

## 1. 语言与编排

| 类别 | 选型 | 说明 |
|------|------|------|
| 语言 | **Python 3** | 主开发语言 |
| 应用框架 / 编排 | **LangChain** | RAG、Agent、Neo4j 集成、Graph 抽取管线 |

---

## 2. 模型与推理

| 用途 | 选型 | 状态 |
|------|------|------|
| Chunk 语义切分（本地） | **Ollama**：`qwen2:1.5b` | 已定 |
| 文本向量（Embedding） | **Ollama**：`mxbai-embed-large:latest` | 已定 |
| 问答 / 抽取 / 路由 / Agent 主 LLM | **`glm-5`** | 已定接入方式：OpenAI 兼容 API（当前项目在 `src/rag_core/config.py` 中硬编码 `openai_base_url/openai_api_key`） |
| 执行形态 | API / 本地等 **按环境与成本选择**，配置项与代码解耦 | 原则 |

---

## 3. 存储与检索

| 类别 | 选型 | 说明 |
|------|------|------|
| 向量数据库 | **Qdrant（本地）** | 语义召回主存储；默认本机 `http://localhost:6333`，本地部署通常无需 API Key |
| 关键词 / 混合检索 | **Elasticsearch（ES）** | 与向量并行，阶段 2 起接入混合检索 |
| 图数据库 | **Neo4j** | 实体关系存储与多跳查询 |
| 重排（Rerank） | **`jina-reranker-v2`** | 阶段 2；具体对接 **Jina API** 或兼容部署以实现为准 |

---

## 4. RAG 与图谱模块

| 模块 | 选型 | 说明 |
|------|------|------|
| 基础 RAG | LangChain + Qdrant | 阶段 1 |
| 检索增强 | Query 改写、多 Query 扩展；HyDE / Step-back **按需** | 阶段 2 |
| 混合检索融合 | 如 **RRF** 或加权 | 阶段 2 |
| 图谱抽取 | **LangChain**：`Neo4jGraph`、`LLMGraphTransformer`；或 **第三方 GraphRAG** 流水线（需对齐 `chunk_id`）；抽取 LLM 默认 **`glm-5`** | 阶段 3 |
| 运行时检索策略 | **方案 1**：向量先行，必要时 **Text-to-Cypher** 查 Neo4j | 阶段 4 |

---

## 5. Agent 与工具

| 类别 | 选型 | 说明 |
|------|------|------|
| Agent 范式 | **ReAct**（Thought → Action → Observation） | 阶段 5 |
| 工具调用 | **Function Calling** 调用检索 / 图查询 | Demo 主路径 |
| Agent 主推理模型 | **`glm-5`**（与上文主 LLM 一致；**降级备用模型**另配） | 阶段 5 |
| 外部工具形态 | **MCP** 与 **自定义 CLI** **二选一待定**；Demo 期可不绑定 | 后续 |
| 联网补全（降级） | **CRAG / 联网搜索** **可选**，本地不足时触发 | 阶段 5 |
| 提示词 | **系统提示词** + 渐进式 **SKILL**（按主文档） | 全局 |

---

## 6. 评测与运维（阶段 6）

| 类别 | 选型 |
|------|------|
| 检索指标 | **Hit@K**、**MRR** |
| RAG 评测框架 | **RAGAs** |
| 文档更新（远期） | 时间戳 / hash；事件驱动可考虑 **Kafka** |
