## 本阶段技术栈

| 类别 | 选型 |
|------|------|
| 图数据库 | **Neo4j** |
| LangChain 集成 | **Neo4jGraph**、**LLMGraphTransformer**（三元组抽取） |
| 抽取用 LLM | **`glm-5`**（默认与主对话一致；若成本高可单独配置轻量模型，须在技术栈约定中注明） |
| 可选路线 | **第三方 GraphRAG**：若采用须映射回自有 **`chunk_id`**，与 Qdrant 对齐 |
| 实体范围（首期） | 疾病、症状、药物 |

## 开发目标

1. 从**与向量库同源的 chunk** 抽取实体与关系，写入 **Neo4j**（chunk 管线以 **01-阶段0** 为准：语义切分 + 重叠策略，保证与向量侧 **chunk_id** 一致）。
2. 使用 LangChain **Neo4jGraph**、**LLMGraphTransformer**（或调研选定的第三方 GraphRAG 流水线），实体类型先收敛为：**疾病、症状、药物**。
3. 节点与边上挂载 **chunk_id / source**，支持回答溯源与增量更新。
4. 若采用第三方自动拆 chunk：约定同步回自有 **chunk_id** 与向量条目，保持「一份文本多处索引」一致。

## 交付物

## 验收要点

- 任意一条边或节点可追溯到对应 chunk_id（或明确标注「来自全文推断」的例外规则）。
- 图谱构建脚本可对新增文档增量运行（不要求阶段 6 级的事件驱动，但要逻辑可行）。

## 依赖与风险

- **依赖**：阶段 0；建议阶段 2 检索基线已稳定，避免频繁改 chunk 切分规则导致图重建。
