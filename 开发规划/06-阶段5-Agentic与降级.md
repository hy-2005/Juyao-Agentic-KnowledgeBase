## 开发目标

1. 采用 **ReAct** 思路：Thought → Action（工具调用）→ Observation → 循环直至终止条件。
2. 工具集：**向量检索**、**图查询** 为必选；可选 **CRAG/联网补全**（本地知识不足或自检不可靠时）。
3. **大模型降级**：超时切换轻量模型、token/轮次上限、API 异常切换备用模型（与主文档一致）。
4. Agent 自身降级：步数上限、连续工具失败则收敛为简短回答。
5. MCP / 定制化 CLI：Demo 期可用 Function Calling 替代；集成方式列入迭代清单。

## 交付物

- 可调的最大步数、工具白名单、超时与重试策略配置。
- 结构化日志：每一步使用的工具类型与关键参数摘要（脱敏）。
- 可选：联网搜索触发条件与免责声明叠加规则。

## 验收要点

- 多跳任务可在有限步数内完成或礼貌终止，不出现无限循环。
- Neo4j/Qdrant 不可用时，降级路径可验证（纯 LLM+联网或极简科普+免责）。

## 依赖与风险

- **风险**：工具过多导致成本与延迟上升；需预算上限与并行策略。
