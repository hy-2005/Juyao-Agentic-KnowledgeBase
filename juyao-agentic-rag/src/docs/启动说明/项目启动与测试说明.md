# juyao-agentic-rag 启动与测试说明

这份文档用于本地快速启动当前 RAG 项目，并完成最小可用测试。

## 1. 进入目录

在仓库根目录执行：

```powershell
cd juyao-agentic-rag
```

## 2. 直接安装依赖（不使用虚拟环境）

```powershell
pip install -e .
```

说明：

- `pip install -e .` 会自动读取 `requirements.txt` 并安装依赖。
- 安装后推荐：`python -m rag_core.agent.run_ingest`、`python -m rag_core.agent.run_qa`。
- `juyao-ingest` / `juyao-rag` 为 setuptools 注册的同名快捷命令，可选使用。
- 如果你后续想和其他 Python 项目隔离，再考虑虚拟环境；当前流程不强制。

## 3. 修改硬编码配置（`config.py`）

打开 `src/rag_core/config.py`，至少确认这些字段：

- `ollama_base_url = "http://localhost:11434"`
- `chunk_model = "qwen2:1.5b"`（语义切分模型）
- `qdrant_url = "http://localhost:6333"`
- `qdrant_api_key = None`（本地 Qdrant 通常留空）
- `embed_model = "mxbai-embed-large:latest"`
- `gen_model = "glm-5"`
- `openai_api_key = "你的密钥"`
- `openai_base_url = "https://api.xstx.info/v1"`

## 4. 启动外部服务

在运行项目前，请确认：

- Ollama 本地服务已启动，并已拉取 embedding 模型（默认 `mxbai-embed-large:latest`）。
- Ollama 本地服务已启动，并已拉取切分模型（默认 `qwen2:1.5b`）与 embedding 模型（默认 `mxbai-embed-large:latest`）。
- Qdrant 本地服务已启动（默认端口 `6333`）。
- 问答模型接口可用（`openai_base_url` + `openai_api_key` 有效，配置于 `config.py`）。

如需手动拉取模型（可选）：

```powershell
ollama pull qwen2:1.5b
ollama pull mxbai-embed-large:latest
```

## 5. 导入样例数据（构建索引）

```powershell
python -m rag_core.agent.run_ingest --file src/data/samples/sample_medical.txt
```

等价别名（可选）：

```powershell
juyao-ingest --file src/data/samples/sample_medical.txt
```

成功后会输出“导入完成，共写入 X 个 chunk”。
首次导入会自动创建 Qdrant 集合 `juyao_knowledge_chunks`，无需手工建库。
当前切分策略为：段落级候选 + 千问小模型语义断点选择 + chunk 前后上下文重叠；不按标点符号强制截断。

## 6. 运行问答测试

```powershell
python -m rag_core.agent.run_qa --question "请简要介绍知识库中关于感冒处理的关键信息"
```

等价别名（可选）：

```powershell
juyao-rag --question "请简要介绍知识库中关于感冒处理的关键信息"
```

预期输出包含：

- 回答正文
- 引用的 `chunk_id`
- 检索分数
- 免责声明

## 7. 快速自检项

- **依赖是否安装成功**
  - `python -m rag_core.agent.run_qa --help` 能正常显示帮助信息
- **配置是否生效**
  - `src/rag_core/config.py` 中地址、模型名、`openai_api_key` 正确
- **索引是否可用**
  - 先执行入库，再执行问答

## 8. 常见问题

- `ModuleNotFoundError: rag_core`
  - 一般是没执行 `pip install -e .`，重新执行一次。

- Qdrant 连接失败
  - 检查 `QDRANT_URL` 是否正确、服务是否启动。

- Ollama 调用失败 / 模型不存在
  - 检查 `ollama_base_url`、`chunk_model`、`embed_model`；先确保切分模型与 embedding 模型都已拉取。

- 问答调用 401 / 403 / 密钥错误
  - 检查 `src/rag_core/config.py` 的 `openai_api_key` 是否有效，`openai_base_url` 是否可访问。

- 检索结果为空
  - 先确认已执行入库命令，且问题与导入文本相关。
- 集合不存在（404）
  - 现在已支持：问答阶段会按“证据不足”返回；执行一次入库会自动创建集合。
