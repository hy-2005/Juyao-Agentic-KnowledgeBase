# 背景
想做一个「能像医生一样顺着关系推理」的智能客服 Demo合理的看病需求，主要是做

医疗知识超级客服。疾病、症状、药物、基因之间有大量关联，如果只用向量检索，「某个基因突变会导致哪些疾病，这些疾病又有哪些共同的治疗方案」这种沿着多层关系链追溯的问题根本无从下手，因为没有一篇文档会把这条完整的链路写在一起。所以想做的就是一个类似最简单的 门诊医生看病功能

1. 到时候依靠文档自身时间和数据库字段判断，如果没有超过数据库就做不更新文档，兜底就是通过全文档hash来看hash值是否变化，到时候的修改这些都是依靠元数据将向量库和图谱删除完，然后重新插入chunk

## Agentic-RAG：

1. 大模型自主调用知识库，看自己拿取知识是否生成足够,可以自己补充自己现要的知识（通过functioncall来调用知识库，MCP具体应该用不上）

核心流程（ReAct框架）：思考（Thought）→ 行动（Action，Function Call调用工具）→ 观察（Observation，工具返回结果）→ 循环（评估结果，决定重试/终止）。直到可以做差不多的解答

## 知识图谱方面：(五种方案看RAG笔记)

技术：LangChain 官方自带完整模块：
- `langchain-community Neo4jGraph`：一键连接、读写 Neo4j，配置和 Spring Boot 连数据库一样简单
- 内置**GraphRAG 自动抽取流水线**：直接给 LLM 喂 Chunk，自动提取「疾病 - 症状 - 药物 - 病因」医疗三元组
- 原生 ReAct Agent、工具调用能力，完美契合你写的 Agentic-RAG 流程
- 对接 Qdrant、ES、重排器都丝滑无缝
总结：
连接neo4j框架：Neo4jGraph
chunk抽取更小的chunk框架： LLMGraphTransformer

想要的完整链路是：
- 原始医疗文档 → 切大 Chunk，存入**向量数据库**（做基础语义召回）
- 同一份 Chunk → 交给 GraphRAG，自动做：实体识别、关系三元组 <疾病 - 症状 - 药物> 抽取
- 把抽出来的精细实体、关系，批量写入 **Neo4j** 知识图谱
- 运行时：向量兜底 + 图谱深挖多层医疗因果 / 关联关系





### 方案 1：「向量先行，图后补深度」
#### 完整标准流程

1. **知识库预处理阶段**
    
    1. 原始长文档、笔记、PDF → 切成文本 Chunk
    2. Chunk 1：存入**向量数据库**（做语义模糊召回）
    3. Chunk 2：喂给 LLM，做实体抽取、关系三元组提取
    4. 把「实体节点、关系边」批量写入 **Neo4j 图数据库**
    
2. **用户提问查询阶段（运行时）**
    
    1. 用户输入自然语言问题
    2. 第一步：**向量检索** → 召回语义相似的 Top N Chunk，打底基础答案
    3. 第二步：LLM 判断：**这个问题要不要深挖复杂关系 / 链路？**
        
        - 不需要：直接用向量结果回答
        - 需要：把用户问题 + 召回上下文，交给 LLM 生成合法 Cypher 语句
        
    4. 第三步：拿着 Cypher 去 Neo4j 执行**精准多跳图遍历**，拿到深层关系数据
    5. 第四步：把「向量文本 + 图谱关系结果」全部交给 LLM，整合输出最终回答
    

**优点**

- 实现门槛最低，不容易翻车
- 语义兜底极强，不怕冷僻模糊问题
- 图谱专门解决向量搞不定的深度推理、路径、网络问题

**缺点**

- 图只做事后增强，不是主角

### 方案 2：「图先行，向量做增强」（进阶高阶玩法）

流程反过来：

1. **预处理阶段**
    
    1. 优先把核心知识、强关系全部结构化存入 Neo4j
    2. 给**每个节点、每条关系**单独生成向量 Embedding，也存入图 / 向量库
    
2. **用户提问阶段**
    
    1. 用户问题 → 先做语义向量匹配，找到最相关的**起始实体节点**
    2. 直接在 Neo4j 里围绕这个节点做**大范围图游走、邻居扩展、链路挖掘**
    3. 把遍历到的一大片子图，整体丢给 LLM 去理解、总结、回答
    

**什么时候用？**

- 社交网络、金融风控、人物 / 企业关系、知识深度问答
- 你就是主打关系推理的场景

### 一、为什么医疗场景优先选【方案 1：向量先行，图后补深度】

医疗问答有 3 个天生特点，完美适配方案 1：

1. **绝大多数普通问诊、科普问题，本质是语义理解**
    
    - 比如：“感冒发烧要注意什么？”“高血压日常怎么饮食？”“这个药有什么副作用？”
    - 纯向量 RAG 召回病历、指南、药典、科普文档，就能给出靠谱回答，速度快、效果稳。
    
2. **复杂医疗关系推理，才需要图谱深度兜底**
    
    只有这类问题，才必须深挖 Neo4j 图谱：
    
    - 疾病 A 会引发哪些并发症 B、C、D？
    - 症状 X+Y+Z，关联哪些疑似疾病？
    - 药品→适应症→禁忌症→搭配冲突、联用风险
    - 患者病史→过往诊断→用药全链路溯源
    
3. **医疗容错率极低，兜底优先级最高**
    
    向量先召回原文权威资料，保证基础回答不出大错；
    
    只有真的需要关系、链路、风险排查时，才触发图查询，避免幻觉、乱推理。

#### 医疗场景下方案 1 完整适配流程

##### 预处理阶段

1. 医疗指南、药典、病历科普、科室资料 → 切 Chunk
2. Chunk 存入向量库：日常科普、症状解释、原文依据检索
3. Chunk 喂 LLM 抽取三元组：`疾病-症状`、`疾病-用药`、`药品-禁忌`、`并发症关联` → 灌入 Neo4j

##### 用户问诊运行阶段

1. 用户输入症状 / 健康问题
2. 向量召回权威医疗原文，打底安全、合规的基础答案
3. LLM 做 ReAct 判断：
    
    - 只是普通解释、科普、注意事项：直接向量结果回答
    - 需要**多症状鉴别、用药冲突、疾病因果、风险链路排查**：自动生成 Cypher 查 Neo4j
    
4. 图谱拿到深层关联、禁忌、风险关系
5. 向量原文 + 图谱精准关系，整合给出严谨、溯源的医疗回复

#### 方案 1 医疗专属优点

- 上手最快，前期快速跑通 Demo、验证效果
- 合规 & 安全拉满：所有回答都有原文出处，降低医疗幻觉
- 可灰度迭代：初期少做图谱，后期慢慢扩充医疗实体库
- 维护成本低：新增医疗资料直接加 Chunk，不用大规模重构图谱

### 二、什么时候用【方案 2：图先行，向量做增强】

**方案 2 更适合成熟、深度垂直的硬核医疗系统**，比如：

- 专科辅助诊断系统
- 慢病全病程管理、患者健康档案追踪
- 药企药品适应症 & 不良反应全网追溯
- 医院科室、医患、诊疗流程网络分析

#### 方案 2 医疗优势

- 核心医疗知识全结构化锁死，推理极其精准
- 围绕疾病、药品实体做全局游走，一次性挖出全部关联风险、并发症、关联检查
- 适合需要强逻辑、强因果、严谨推导的专业医疗场景

#### 缺点（新手大坑）

- 前期图谱构建成本爆炸，医疗实体、关系极其复杂，拆分极容易出错
- 冷启动极难，一开始图谱不全，效果拉胯
- 口语化、普通用户提问适配差

### 三、医疗项目落地最优路线（保姆级建议）

1. **第 1 阶段（现在起步）**
    
    直接用**方案 1**，快速搭建：
    
    - 先把公开医疗知识库、权威科普、诊疗指南跑通基础向量 RAG 问答
    - 只先抽最核心的疾病 - 症状 - 药品三类三元组进 Neo4j
    - 实现「普通问题向量答、复杂关系问题走图谱」
    
2. **第 2 阶段（迭代优化）**
    
    效果稳定后，向**混合方案**升级：
    
    - 给 Neo4j 里的医疗实体批量加向量 Embedding
    - 支持：既可以向量引路查图，也可以图深挖后再召回补充原文
    
3. **强制医疗红线提醒**
    
    - 所有 AI 回答开头必须标注：**仅科普参考，不能替代专业医生诊断，务必线下就医**
    - 禁止 AI 给出确诊、开药等诊疗行为
    - 全程溯源引用权威医疗来源，极大降低风险

## 降级方案
### 大模型调用降级
1. 超时降级
2. token数降级
3. 轮次降级

### Agentic-RAG RAG 图数据库查不到

- **主链路**：本地向量库 + Neo4j 知识图谱 → 优先用内部知识回答
- **降级策略**：当本地知识不够、回答不靠谱、出现幻觉时→ **触发 CRAG 模块 → 自动走联网搜索补全信息**

###  完整多级降级方案

1. **大模型层降级**
    
    - 超时降级：复杂重推理大模型超时，自动切换轻量化快速模型
    - Token 超限降级：上下文过长，压缩文档与对话后降级模型规格
    - 限流 / 异常降级：API 报错、额度用尽，切换备用模型 / 兜底开源模型
    
2. **知识链路智能降级（核心）**
    
    - 最优通路：向量召回 + Neo4j 知识图谱深度推理回答
    - 图谱失败降级：图查询无结果 / 报错 → 降级为纯向量 RAG 作答
    - 召回质量降级：相似度不足、内容可信度低 → 触发 CRAG 纠错校验
    - CRAG 兜底降级：自检发现幻觉、信息缺失 → 自动调用联网搜索补全权威信息
    - 终极保底：全链路检索失效 → 输出保守科普 + 强制免责声明
    
3. **Agent 执行降级**
    
    - ReAct 思考循环超限、工具调用频繁失败 → 终止深度推理，基于已有信息极简输出
    
4. **数据库服务降级**
    
    - Neo4j / 向量库服务异常、连接失败 → 跳过专属库，直接走「LLM + 联网」极简可用模式
    
5. **医疗安全合规降级**
    
    - 触及诊断、开药等高风险诉求 → 强制拦截，仅提供健康科普，严禁医疗指导
    - 溯源可信度不足 → 弱化断言，强化免责与线下就医引导
